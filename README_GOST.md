# Руководство по Gostdown Automation

## Обзор
Система позволяет генерировать отчеты ГОСТ 7.32 из Markdown, используя гибридную среду Linux+Windows.

## Структура папок
- `input/`: Папка для входных проектов.
  - `input/MyReport/`: Папка проекта.
    - `main.md`: Основной текст (или набор .md файлов).
    - `reference.docx`: Шаблон стилей (обязательно).
- `output/`: Сюда сохраняются результаты.
- `Makefile`: Управление сборкой.
- `gostdown_runner.py`: Скрипт-обертка.

## Как использовать

### 1. Подготовка
Создайте папку проекта в `input/`. Например `input/Lab1`.
Положите туда:
- Markdown файлы (`.md`). Они будут склеены в алфавитном порядке.
- `reference.docx` — файл с нужными стилями (можно взять из `gostdown/demo-template-report.docx`).

### 2. Запуск
Из папки `/mnt/win10-share/gostdown-proj` на Linux:

```bash
# Сборка всех проектов
make all

# Параллельная сборка (всех ядер)
make -j$(nproc) all

# Сборка конкретного проекта
make output/Lab1/Lab1.docx
```

### 3. Настройка Mermaid
Поддержка диаграмм включена по умолчанию в Makefile (`--mermaid`).
Для работы на Windows VM должен быть установлен `mermaid-filter`.
В markdown используйте:

    ```mermaid
    graph TD;
        A-->B;
    ```

### 4. Особенности
- **Многопоточность**: Pandoc работает в один поток на один проект. Для ускорения запускайте сборку нескольких отчетов одновременно через `make -j`.
- **Пути**: Все файлы должны лежать внутри `/mnt/win10-share`.
